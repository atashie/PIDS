# Reviews (Adversarial Verification Outputs)

Raw outputs of verification work — human re-derivations and agent skeptic runs — that back the status of each claim in [`../claims-register.md`](../claims-register.md).

## Naming convention
`<claim-id>__<method>__<YYYY-MM-DD>.md`

Examples:
- `C-001__darcy-rederivation__2026-06-15.md`
- `C-002__literature-K-comparison__2026-06-15.md`
- `C-006__agent-refutation-panel__2026-06-16.md`

## What each review file should contain
1. The claim under test (ID + statement).
2. The stance: *attempt to refute*.
3. Method & inputs (sources used, calculations, assumptions).
4. Result: refuted / not refuted, with reasoning.
5. Recommended status change for the register.

See [`../../governance/verification-protocol.md`](../../governance/verification-protocol.md) for the rules and `../../tools/review-agents/` for agent templates.
